GitHub Copilot CLI 說明

最後更新：2025-11-17T06:31:48.993Z · 本頁根據 GitHub Copilot CLI 官方文件與內建說明彙整。

什麼是 Copilot CLI？

GitHub Copilot CLI 是一個互動式終端助理，可在命令列中協助你理解程式碼、生成命令、編輯檔案與自動化常見工作流程。

它提供內建的指令（/commands）與快捷鍵，並可依你的偏好調整主題、模型與檔案可讀取範圍。

快速開始

  1. 執行 /login 登入 GitHub Copilot。
  2. 使用 /help 檢視互動式指令與快捷鍵。
  3. 用 /cwd 檢視或切換目前工作目錄；用 /theme 切換外觀；用 /model 選擇模型。
  4. 完成後以 /exit 或 /quit 離開。

安裝方式與最新需求請參考下方「更多資源」。

常用快捷鍵

全域

  • @：提及檔案，將內容納入目前脈絡
  • Esc：取消目前操作
  • !：在本機殼層直接執行指令（不送給 Copilot）
  • Ctrl+c：思考中取消／清除輸入／離開
  • Ctrl+d：關閉
  • Ctrl+l：清除畫面

展開時間軸

  • Ctrl+o：展開/收合全部時間軸
  • Ctrl+r：展開/收合最近時間軸

游標/編輯

  • Ctrl+a/e：移至行首/行尾
  • Ctrl+h：刪除前一字元
  • Ctrl+w：刪除前一個字
  • Ctrl+u/k：刪除至行首/行尾
  • Alt+←/→：以字為單位移動
  • ↑/↓：瀏覽歷史指令

內建互動式指令

  • /add-dir <directory>：允許 Copilot 讀取指定目錄
  • /agent：瀏覽並選擇可用的專家代理（若有）
  • /clear：清除對話歷史
  • /cwd [directory]：切換或顯示目前工作目錄
  • /delegate <prompt>：以 AI 生成 PR 的方式委派變更
  • /exit、/quit：離開 CLI
  • /feedback：提供意見回饋
  • /help：顯示互動式指令說明
  • /list-dirs：列出已允許讀取的目錄
  • /login、/logout：登入/登出 Copilot
  • /mcp [show|add|edit|delete|disable|enable] [server-name]：管理 MCP 伺服器設定
  • /model [model]：選擇使用的 AI 模型
  • /reset-allowed-tools：重設允許使用的工具清單
  • /session：顯示目前工作階段資訊
  • /terminal-setup：設定多行輸入（Shift+Enter / Ctrl+Enter）
  • /theme [show|set|list] [auto|dark|light]：檢視/設定主題
  • /usage：顯示使用統計
  • /user [show|list|switch]：管理 GitHub 使用者

遵循的指示來源

Copilot CLI 會遵循以下位置的指示檔：

  • .github/instructions/**/*.instructions.md（git root 與目前目錄）
  • .github/copilot-instructions.md
  • AGENTS.md（git root 與目前目錄）
  • CLAUDE.md、GEMINI.md
  • %HOMEPATH%/.copilot/copilot-instructions.md
  • 以及由 COPILOT_CUSTOM_INSTRUCTIONS_DIRS 指定的其他目錄

使用小技巧

  • 在提出複雜請求前，先用 /help 熟悉可用指令與限制。
  • 管理可讀取的目錄（/add-dir、/list-dirs），讓 Copilot 取得必要的檔案脈絡。
  • 需要自訂整合時，透過 /mcp 管理 MCP 伺服器設定。
